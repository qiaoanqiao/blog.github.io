Docker镜像下载\删除\查看命令_参数详解

标签（空格分隔）： docker

镜像 下载\删除\查看:

  1. 镜像 下载:

    1) . 下载最新系统

    这里的centos是系统,如果后面不指定系统详细版本号会下载镜像源最新系统 *
    　　
    2).*    下载指定版本系统**

    下载不同版本在centos后面加上冒号然后是版本号
    　　
    后面使用的版本号依据搜索出来的镜像提供的一些版本编号来填写
    　　

    3).Docker常用搜索镜像下载地址:
    Docker官网的镜像搜索
    阿里云提供的国内镜像下载 　　

  2. 镜像 删除\查看:

    1.停止所有的经过镜像实例化运行的容器，这样才能够删除其中的images：
    　　
    查看容器:

    options
    名字 简写 默认 描述
    –time -t 10 秒杀之前等待停止

    3.删除images，通过image的id来指定删除谁

    例子:

$ docker images

REPOSITORY                TAG                 IMAGE ID            CREATED             SIZE
test1                     latest              fd484f19954f        23 seconds ago      7 B (virtual 4.964 MB)
test                      latest              fd484f19954f        23 seconds ago      7 B (virtual 4.964 MB)
test2                     latest              fd484f19954f        23 seconds ago      7 B (virtual 4.964 MB)

$ docker rmi fd484f19954f

Error: Conflict, cannot delete image fd484f19954f because it is tagged in multiple repositories, use -f to force
2013/12/11 05:47:16 Error: failed to remove one or more images

$ docker rmi test1

Untagged: test1:latest

$ docker rmi test2

Untagged: test2:latest


$ docker images

REPOSITORY                TAG                 IMAGE ID            CREATED             SIZE
test                      latest              fd484f19954f        23 seconds ago      7 B (virtual 4.964 MB)

$ docker rmi test

Untagged: test:latest
Deleted: fd484f19954f4920da7ff372b5067f5b7ddb2fd3830cecd17b96ea9e286ba5b8

#### 问题回顾 　　 删除时出现 　　

　　
可以尝试
　　
docker rmi daocloud.io/centos #REPOSITORY 栏中的文字
　　
再次删除时出现
　　
Untagged: daocloud.io/centos:latest
　　
尝试删除
　　
docker rmi #IMAGE ID 栏中的文字
